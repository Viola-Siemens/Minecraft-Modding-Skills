# 审查流程

根据 Minecraft EULA、使用指南及中国相关法律法规，对 Minecraft 模组（Mod）的代码、资源、文档及发布包进行全面的合规性审查，识别违规内容与潜在风险，并给出具体的修正建议。

## 适用场景

* 模组开发者准备发布前的自查
* 社区/平台（如 CurseForge、Modrinth）的内容审核辅助
* 整合包作者对收录模组的二次审查

## 输入内容

探索项目上下文，了解以下内容：

* 模组名称、版本、依赖的模组加载器（Forge/Fabric 等）
* 文件列表（jar 内文件结构、发布包所含文件），通常可通过 `build.gradle` 构建脚本推断
* 代码片段（如有，可提供关键部分进行安全审计）
* 是否使用第三方素材清单及其授权证明
* README、LICENSE、隐私政策等文档内容
* 是否存在联网功能、用户生成内容入口、虚拟货币等说明
* 模组说明文字、宣传文案（如果支持多模态输入，也可以了解宣传截图和视频链接）

如果项目中的文件不足以体现上述内容，请向用户提问，以收集足够的信息判断其合规性。

## 输出格式

生成一份合规审查报告，包含：

1. 总体结论：通过 / 有条件通过 / 不通过
2. 高优先级问题清单（必须修改）
3. 中优先级问题清单（强烈建议修改）
4. 低优先级建议清单（完善性建议）
5. 每项问题的详细说明：违规点、法律/协议依据、修正方案

# 检查清单

审查时需按以下六个维度逐一核对，依据“优先级”判断严重性，并结合“检查要点”给出审查结论。

## 1. 禁止内容审查（一票否决）

| 检查项 | 检查要点 | 违规示例 | 修正建议 | 优先级 |
|---|---|---|---|---|
| 淫秽/色情内容 | 文本、图片、音频中是否含有 R18、露骨性暗示等 | 皮肤贴图裸露敏感部位，对话含性描写 | 删除或替换违规素材，加入全年龄向过滤 | 高 |
| 仇恨/歧视言论 | 游戏内文本、脚本、NPC 对话等（亦包括游戏日志和异常消息）是否包含种族、性别、地域、宗教歧视 | 物品名称含侮辱性词汇 | 替换为中性表达，清除歧视内容 | 高 |
| 严重暴力/恐怖 | 视觉素材是否过度血腥、恐怖或引发严重不适 | 尸体细节特写，惊悚跳吓 | 降低血腥程度，添加内容警告标签 | 高 |
| 违法内容暗示 | 有无涉及非法交易、毒品、赌博、极端政治宣传等情节 | 添加“赌场”小游戏，毒品制作配方 | 完全移除相关玩法和文本 | 高 |
| 恶意代码/作弊 | 是否捆绑木马、病毒、作弊工具，是否绕过正版验证或篡改客户端 | jar 内含键盘记录器，强制解锁正版功能 | 逐行审计代码，删除恶意或无关可执行文件 | 高 |
| 赌博/博彩机制 | 是否含真实货币或虚拟货币押注、开箱抽奖（未公示概率）等 | 充值抽奖未公布概率 | 移除博彩元素，或改为纯免费且明示概率 | 高 |

## 2. 许可与分发限制审查

| 检查项 | 检查要点 | 修正建议 | 优先级 |
|---|---|---|---|
| 禁止售卖模组/功能 | 模组本体或核心功能是否收费，是否存在“付费赢”道具 | 取消所有内购与强制收费，仅保留自愿赞助/捐赠渠道 | 高 |
| 分发内容合规 | 发布包是否包含 Minecraft 原版 jar、资源文件、完整客户端 | 确保仅打包模组自身文件，移除一切 Mojang 版权资产 | 高 |
| 商业素材授权 | 第三方模型、贴图、音频、字体是否有可商用许可证 | 替换为开源/已授权素材，补齐授权声明文件 | 高 |
| 原版内容修改限制 | 是否直接修改或重分发 Minecraft 源代码、打包“Mod 化客户端” | 改为以独立模组加载器（Forge/Fabric）方式运行，删除原版文件 | 高 |
| 开源许可与声明 | 是否保留第三方开源代码的原始许可证，自身 LICENSE 文件是否缺失 | 补充所有版权与许可声明，包含 MIT/GPL 等原始文本 | 中 |

## 3. 技术合规审查

| 检查项 | 检查要点 | 修正建议 | 优先级 |
|---|---|---|---|
| 游戏公平性与完整性 | 是否包含破坏多人游戏平衡的功能（自动化攻击、飞行外挂、透视等） | 移除作弊性质功能，或在单机模式下限制启用 | 中 |
| 反 DRM 与破解 | 是否绕过官方启动器或验证，是否需盗版客户端支持 | 只兼容正版环境，去除所有破解补丁 | 高 |
| 性能与兼容性 | 是否注明支持的 MC 版本、加载器，有无明显性能缺陷（内存泄漏、死循环） | 提供兼容性列表，优化代码，标注最低配置 | 中 |
| 安全漏洞扫描 | 联网功能是否存在输入校验缺失、权限绕过、RCE 等风险 | 进行手动审计或使用静态分析工具，修补所有高危漏洞 | 中 |

## 4. 第三方素材合规审查（中优先级）

- **字体**：检查字体文件授权是否为免费商用（如 SIL OFL），如仅个人免费则需更换。
- **音频**：音乐、音效需有 CC0、CC BY 等可商用许可，并按要求署名。
- **模型/贴图**：确认无商业版权角色（如宝可梦、漫威），素材许可证齐全，并在发布包内附带“第三方素材授权说明”。
- **商标使用**：宣传材料中不得将“Minecraft”“Mojang”作为主体名称使用，允许以“XXX (Minecraft Mod)”形式说明，且不得擅自使用官方 Logo。

## 5. 用户生成内容与社区管理审查（中优先级）

- **举报与反馈渠道**：是否在发布页或文档中提供有效联系邮箱/论坛链接。
- **未成年人保护（中国地区）**：若存在聊天、上传功能，需有关键词过滤和健康游戏提醒；不得诱导未成年人付费或过度游戏。
- **社区规则**：是否附有简明行为准则，禁止骚扰、歧视，并说明违规处理机制。

## 6. 法律与发布文档审查

| 检查项 | 检查要点 | 修正建议 | 优先级 |
|---|---|---|---|
| 隐私与数据保护 | 模组是否收集玩家数据（ID、IP 等），有无隐私政策，是否符合《个人信息保护法》 | 如收集信息必须明文告知并获同意，提供隐私设置选项 | 高 |
| 未成年人网络保护（中国地区） | 是否适配防沉迷要求，有无适龄提示 | 添加游戏时长提醒，限制未成年人敏感功能 | 低 |
| 出口管制与金融法规 | 是否包含加密算法或虚拟货币兑换系统 | 避免涉及真实金融交易，合规使用加密技术 | 中 |
| 安装文档与许可 | README 是否含安装步骤、依赖列表，自身许可证与第三方版权声明是否完整 | 补充清晰文档，放置 LICENSE 文件 | 低 |
| 联系方式 | 是否公开维护者联系方式 | 添加邮箱、漏洞跟踪器、仓库链接或Discord 链接，保证可联系 | 低 |
